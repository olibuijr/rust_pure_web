Creating Your First Page

Create a new HTML template in the public/templates/ directory:

1 Create the Template

Create public/templates/hello.html:

{% layout "layouts/root.html" %}

<div class="min-h-screen flex items-center justify-center">
    <div class="text-center">
        <h1 class="text-4xl font-bold mb-4">Hello, {{ name }}!</h1>
        <p class="text-muted-foreground">Welcome to Rust Pure Web</p>
    </div>
</div>

2 Add the Route

Open src/pages.rs and add a new page struct and route:

pub struct HelloPage {
    pub name: String,
}

impl HelloPage {
    pub fn render(&self) -> String {
        let mut ctx = settings_context("Hello");
        ctx.set("name", &self.name);

        let content = template::load("hello.html");
        template::render(&content, &ctx)
    }
}

Template Engine Basics

The template engine supports Jinja2-style syntax:

Variables

Output values with double braces:

<h1>Hello, {{ username }}</h1>
<p>Your email is {{ email }}</p>

Includes

Include reusable components:

{% include "components/nav.html" %}
{% include "components/footer.html" %}

Layouts

Extend a parent layout (Next.js-style):

{% layout "layouts/root.html" %}

<!-- Your page content here -->
<!-- It will be injected into {{ children }} of the layout -->

Creating a Collection via API

Use the REST API to create data collections. First, authenticate to get a token: 

# Login and get token
curl -X POST http://localhost:3460/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@example.com", "password": "your-password"}'

# Response: {"token": "your-auth-token", ...}

Then create a collection with the token: 

# Create a "posts" collection
curl -X POST http://localhost:3460/api/collections \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "posts",
    "fields": [
      {"name": "title", "type": "string"},
      {"name": "content", "type": "string"},
      {"name": "published", "type": "bool"}
    ]
  }'

# Add a document to the collection
curl -X POST http://localhost:3460/api/collections/posts \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title": "My First Post", "content": "Hello World!", "published": true}'

Hot Reload During Development

Templates automatically reload when you save changes. No server restart needed.

Edit any .html file in public/templates/
Save the file
Browser refreshes automatically with your changes

Note on Rust Changes

Changes to Rust code (in src/) require a server restart. Only template changes benefit from hot reload.
