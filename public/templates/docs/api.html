API Reference

Complete REST API documentation for all endpoints.

Base URL

http://localhost:3460/api

All API requests return JSON. Authentication uses Bearer tokens in the Authorization header.

Authentication Endpoints

POST /api/auth/register Create account
POST /api/auth/login Get token
GET /api/auth/me Current user
POST /api/auth/logout End session

Collection Endpoints

All collection endpoints require authentication.

GET /api/collections List all collections
POST /api/collections Create collection (admin)
DELETE /api/collections/:name Delete collection (admin)

Document Endpoints

CRUD operations on documents within a collection.

GET /api/collections/:name List documents
POST /api/collections/:name Create document
GET /api/collections/:name/:id Get document
PUT /api/collections/:name/:id Update document
DELETE /api/collections/:name/:id Delete document

Public Endpoints

No authentication required.

POST /api/contact Submit contact form

Admin Endpoints

Requires admin role.

GET /api/admin/stats Get statistics
POST /api/admin/backup Create backup
GET /api/admin/users List all users
POST /api/admin/users Create user
PUT /api/admin/users/:id Update user
DELETE /api/admin/users/:id Delete user
GET /api/admin/settings Get settings
PUT /api/admin/settings Update settings
GET /api/admin/collections/system List system collections
POST /api/admin/chat Ollama AI chat

Example: Create Collection

curl -X POST http://localhost:3460/api/collections \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "posts",
    "fields": [
      {"name": "title", "type": "string"},
      {"name": "content", "type": "string"},
      {"name": "published", "type": "bool"}
    ]
  }'

Example: CRUD Operations

# Create
curl -X POST http://localhost:3460/api/collections/posts \
  -H "Authorization: Bearer TOKEN" \
  -d '{"title": "Hello", "content": "World"}'

# Read all
curl http://localhost:3460/api/collections/posts

# Read one
curl http://localhost:3460/api/collections/posts/DOCUMENT_ID

# Update
curl -X PUT http://localhost:3460/api/collections/posts/DOCUMENT_ID \
  -H "Authorization: Bearer TOKEN" \
  -d '{"title": "Updated Title"}'

# Delete
curl -X DELETE http://localhost:3460/api/collections/posts/DOCUMENT_ID \
  -H "Authorization: Bearer TOKEN"

Example: Contact Form

# Submit contact form (no auth required)
curl -X POST http://localhost:3460/api/contact \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "message": "Hello, I have a question..."
  }'

Example: Admin User Management

# List all users
curl http://localhost:3460/api/admin/users \
  -H "Authorization: Bearer ADMIN_TOKEN"

# Create a new user
curl -X POST http://localhost:3460/api/admin/users \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "newuser@example.com",
    "password": "secure-password",
    "role": "user"
  }'

# Update user
curl -X PUT http://localhost:3460/api/admin/users/USER_ID \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"role": "admin"}'

# Delete user
curl -X DELETE http://localhost:3460/api/admin/users/USER_ID \
  -H "Authorization: Bearer ADMIN_TOKEN"

Example: Admin Settings

# Get current settings
curl http://localhost:3460/api/admin/settings \
  -H "Authorization: Bearer ADMIN_TOKEN"

# Update settings
curl -X PUT http://localhost:3460/api/admin/settings \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "site_name": "My App",
    "maintenance_mode": false
  }'

# Get system collections
curl http://localhost:3460/api/admin/collections/system \
  -H "Authorization: Bearer ADMIN_TOKEN"

Example: Ollama AI Chat

# Send chat message to Ollama
curl -X POST http://localhost:3460/api/admin/chat \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Explain how authentication works",
    "model": "llama3.2"
  }'

Requires Ollama running locally. Default model can be configured in settings.

Configuration

The OLLAMA_HOST environment variable must be set for this endpoint to work.

OLLAMA_HOST=http://localhost:11434

This is optional and only needed if using the AI chat feature.

Error Responses

400 Bad Request - Invalid input or validation error
401 Unauthorized - Missing or invalid token
404 Not Found - Resource doesn't exist
