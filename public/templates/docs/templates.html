Template Engine

Jinja2-style templates with Next.js-style layouts. 79 lines of code.

Variables Loops Layouts Components

Overview

The template engine supports Jinja2 syntax with a Next.js-style layout system. Props flow through nested layouts via Context, enabling clean separation of concerns.

Basic Syntax

Variables

{{ name }}
{{ page_title }}

Includes

{% include "components/nav.html" %}
{% include "components/footer.html" %}

Loops

{% for skill in skills %}
    <div>{{ skill.name }}</div>
{% endfor %}

Conditionals

{% if available %}
    <span>Available</span>
{% else %}
    <span>Busy</span>
{% endif %}

Layout System

Next.js-style nested layouts. Each layout can wrap content and extend a parent layout.

1 Root Layout

Base HTML structure. Uses {{ children }} for content.

<!-- layouts/root.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{{ page_title }}</title>
</head>
<body>
    {{ children }}
</body>
</html>

2 Nested Layout

Extends root with {% layout %}. Adds sidebar, nav, etc.

<!-- layouts/docs.html -->
{% layout "layouts/root.html" %}

<nav>Documentation</nav>
<aside>Sidebar</aside>
<main>
    {{ children }}
</main>

3 Page Content

Pages specify their layout. Content goes into {{ children }}.

<!-- docs/api.html -->
{% layout "layouts/docs.html" %}

<h1>API Reference</h1>
<p>This content appears in the main area.</p>

Context (Props)

Props flow through all layouts. Set once, available everywhere. 

let mut ctx = Context::new();
ctx.set("page_title", "API Reference");
ctx.set("active_api", "text-foreground font-medium");

// Props available in root, docs layout, AND page content
template::render(&content, &ctx);

Modular Components

Keep pages minimal by extracting reusable pieces into components.

Component Structure

components/
├── nav.html           # Site navigation
├── footer.html        # Site footer
└── admin/             # Admin panel components
    ├── login.html
    ├── nav.html
    ├── sidebar.html
    ├── dashboard.html
    ├── collections.html
    ├── api-reference.html
    └── scripts.html

Creating a Component

<!-- components/admin/nav.html -->
<nav class="border-b">
    <span>Admin Panel</span>
    <button onclick="logout()">Logout</button>
</nav>

Composing a Page

The admin.html is just 31 lines:

<!DOCTYPE html>
<html>
<body>
    {% include "components/admin/login.html" %}
    <div id="admin-panel">
        {% include "components/admin/nav.html" %}
        {% include "components/admin/sidebar.html" %}
        {% include "components/admin/dashboard.html" %}
        {% include "components/admin/collections.html" %}
    </div>
        {% include "components/scripts/admin.html" %}
</body>
</html>

Minimal Implementation

The entire template engine is 79 lines of Rust. Core functions: load(), render(), process_includes(), and Context.
