Cryptography

Pure Rust implementations of industry-standard cryptographic primitives.

SHA-256 HMAC PBKDF2 ChaCha20

Overview

All cryptographic operations are implemented in pure Rust without any external dependencies. Each algorithm follows the official NIST and RFC specifications exactly, ensuring compatibility and security. No unsafe code is used.

SHA-256

FIPS 180-4

Secure Hash Algorithm producing 256-bit (32-byte) digests. Used as the foundation for HMAC and PBKDF2.

Rust API

pub fn sha256(data: &[u8]) -> [u8; 32]

Properties

  • 64 rounds of compression
  • 512-bit blocks with padding
  • Big-endian byte order
üîë

HMAC-SHA256

RFC 2104

Hash-based Message Authentication Code. Provides both data integrity and authentication using a secret key.

Rust API

pub fn hmac_sha256(
    key: &[u8],
    data: &[u8]
) -> [u8; 32]

Properties

  • 64-byte block size
  • Keys longer than 64 bytes are hashed
  • Uses ipad (0x36) and opad (0x5c)
üîê

PBKDF2-SHA256

RFC 8018

Password-Based Key Derivation Function 2. Stretches passwords into secure keys using many iterations to resist brute-force attacks.

Rust API

pub fn pbkdf2(
    password: &[u8],
    salt: &[u8],
    iterations: u32
) -> [u8; 32]

Configuration

  • 100,000 iterations (default)
  • 16-byte random salt
  • 32-byte derived key

Password Storage Format

salt_hex:hash_hex (total 128 characters)

ChaCha20

RFC 8439

Stream cipher designed by Daniel J. Bernstein. Fast, secure, and resistant to timing attacks. Used for database encryption.

Rust API

pub fn chacha20(
    key: &[u8; 32],
    nonce: &[u8; 12],
    data: &[u8]
) -> Vec<u8>

Properties

  • 256-bit key
  • 96-bit nonce
  • 20 rounds (quarter rounds)
  • XOR-based (encrypt = decrypt)

Helper Functions

hash_password

Generates a random salt and hashes the password using PBKDF2-SHA256.

// Returns: "salt_hex:hash_hex"

verify_password

Verifies a password against a stored hash. Uses constant-time comparison. 

pub fn verify_password(
    password: &str,
    stored: &str
) -> bool

random_bytes

Generates cryptographically secure random bytes from /dev/urandom. 

pub fn random_bytes(len: usize) -> Vec<u8>

Usage in the System

1

Password Hashing

User passwords are hashed with PBKDF2-SHA256 (100k iterations) before storage.

2

Session Tokens

32 random bytes from /dev/urandom, hex-encoded to 64 characters.

3

Database Encryption

All data synced to disk is encrypted with ChaCha20 using your SECRET_KEY.

4

Key Derivation

SECRET_KEY is hashed with SHA-256 to derive the 32-byte encryption key.

Security Guarantees

  • All implementations follow official specifications exactly
  • No unsafe Rust code is used
  • Constant-time comparison for password verification
  • Random nonces generated for each encryption operation
  • Keys are never logged or exposed in error messages

Important Notes

  • Keep your SECRET_KEY in .env.local secure
  • If you lose the SECRET_KEY, encrypted data cannot be recovered
  • Use at least 32 characters for the SECRET_KEY
  • The crypto module has no external dependencies
